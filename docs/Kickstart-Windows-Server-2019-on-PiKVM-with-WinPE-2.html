Step 2.1 - PiKVM Test Authentication

Connect via ssh:

ssh {linuxattuneuser}@{attuneosbuildserver}

Execute the following script: 

                
CURL_OUTPUT=$(curl --insecure \
    --header X-KVMD-User:{pikvmAdmin.user} \
    --header X-KVMD-Passwd:{pikvmAdmin.password} \
    https://{pikvmIpAddress}/api/auth/check)
    
echo "${CURL_OUTPUT}" | jq '.'

OK=$(echo "${CURL_OUTPUT}" | jq '.ok')

if [ "${OK}" != "true" ]
then
    echo "Error"
    exit 1
fi

Step 2.9 - PiKVM Upload MSD Image

Execute the following script: 

                
cd {ksAttuneBaseDir}

curl --insecure --verbose \
    --request POST \
    --data-binary @kickstart_{targetServer.fqn}.iso \
    --header X-KVMD-User:{pikvmAdmin.user} \
    --header X-KVMD-Passwd:{pikvmAdmin.password} \
    https://{pikvmIpAddress}/api/msd/write?image=kickstart_{targetServer.fqn}.iso

# Check if this is automatically created 
# /var/lib/kvmd/msd/.__kickstart_{targetServer.fqn}.iso.complete

Step 2.13 - PiKVM Simulate ESC Keypress

Execute the following script: 

                
for _ in $(seq 1 1 45)
do

    curl --insecure \
        --request POST \
        --header X-KVMD-User:{pikvmAdmin.user} \
        --header X-KVMD-Passwd:{pikvmAdmin.password} \
        "https://{pikvmIpAddress}/api/hid/events/send_key?key=End"
        
    sleep 1
    
done

Step 2.14 - KS Win WinRM HTTPS Wait 120s

Check if TCP Port 0 is listening. When the TCP Port is up, wait 120 seconds.

Use Telnet to check if the TCP Service is accepting connections.
